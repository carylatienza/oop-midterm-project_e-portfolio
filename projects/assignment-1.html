Introduction to Java

These are my answers for the assignment.

1. What is Java programming language? Discuss its Features & advantages.

Java is a high-level programming language and computing platform known for its principle of "write once, run anywhere" (WORA). It was developed by Sun Microsystems in 1995 and later acquired by Oracle Corporation. Java is primarily object-oriented, meaning programs are structured around objects and classes. This makes it highly suitable for large-scale systems, enterprise applications, mobile development (especially Android), and back-end systems.


Key Features of Java:

  • Object-Oriented: Java is built around OOP principles such as encapsulation, inheritance, polymorphism, and abstraction.
  • Platform Independent: Programs are compiled into bytecode, which can run on any system with a Java Virtual Machine (JVM).
  • Simple: Java removed complex features from C and C++, such as pointer arithmetic and manual memory management.
  • Secure: Provides built-in security features like no direct pointer manipulation, bytecode verification, and a security manager to prevent unauthorized access.
  • Robust: Features strong memory management and exception handling to reduce crashes and runtime errors.
  • Multithreaded: Supports running multiple tasks simultaneously.


Advantages of Java:

  • Cross-platform compatibility
  • Large community and documentation.
  • Used extensively in Android development.
  • Strong security features and scalability for enterprise systems.

2. Explain the concept of platform independence in Java. How does Java achieve it?

Platform independence means a program can run on different operating systems (Windows, macOS, Linux) without modification. While traditional languages like C compile code directly into machine language specific to an OS, Java uses a two-step process:

  1. Source Code (.java) is compiled into Bytecode (.class).
  2. Bytecode is executed by the JVM.

The JVM acts as an intermediary between the bytecode and the operating system. Because each OS has its own specific JVM implementation, the same bytecode can run anywhere. Therefore, Java is platform-independent, but the JVM itself is platform-dependent.

3. Discuss the main components of a Java program and their roles.

  • Class: Everything in Java is inside a class; it acts as a blueprint.
  • Main Method: The entry point of every application (public static void main(String[] args)).
  • Methods: These define the behavior of objects.
  • Variables: Used to store data (Instance, Local, and Static variables).
  • Objects: Instances of classes that allow access to methods and variables.

4. What are the differences between JDK, JRE, and JVM?

  • JVM (Java Virtual Machine): The engine that executes bytecode and converts it into machine code for the OS.
  • JRE (Java Runtime Environment): Includes the JVM, core libraries, and supporting files; it allows users to run Java programs.
  • JDK (Java Development Kit): Includes the JRE, compiler (javac), and debugging tools; it is used to develop and run Java programs.

5. Is Java an interpreted or compiled language?

Java uses a hybrid model:

  • It is compiled into bytecode using the Java compiler (javac).
  • The bytecode is then interpreted (or JIT-compiled) by the JVM.

Modern JVMs use Just-In-Time (JIT) compilation to improve performance by converting frequently used bytecode into native machine code at runtime.

Reflection

What I learned

Transitioning from C to Java has been a major vibe shift because moving away from manual memory management and pointer arithmetic feels way more structured and disciplined. It is honestly a win for long-term projects since Java favors a clean architecture that helps me stay organized. Learning how "write once, run anywhere" actually works was a lowkey lightbulb moment, especially seeing how the bytecode runs on the JVM middleman instead of talking to my OS directly. It makes so much sense why Java uses a hybrid model to balance performance and portability, and it definitely makes me appreciate why it's the standard for the enterprise systems and Android apps I'm looking into.
