Activity 4 — Student Payment System with Validation Counter

This activity involves creating a menu-driven Java application that simulates tuition fee management, including payments, balance tracking, and one-time scholarship discounts.

Java VS Code Switch Statement Do-While Loop Nested-If Validation Counters

Overview

What this activity covers

The activity focuses on using a do-while loop for program repetition and a switch statement for menu navigation. It also emphasizes nested if statements for input validation and the use of counter variables to monitor student transactions.

Source Code & Output

Reflection

What I learned

Developing this student payment system was an interesting challenge in maintaining data integrity, particularly when handling the logic for the one-time scholarship discount and preventing negative balances. It really made me think about the sequence of operations—like ensuring the 20% discount is applied correctly before subsequent payments and using a counter to track every interaction within the do-while loop. Coming from my background in C, I appreciate how Java’s structure makes it easier to organize these different payment rules into a clean, menu-based system using switch statements. It was satisfying to see the final output correctly summarize the student’s name, total transactions, and final balance, which really reinforces how important disciplined validation is for building reliable software. Verifying that the program handles scenarios like a user trying to pay more than their remaining balance definitely helped me understand the practical side of Java's robust logic and control flow.
