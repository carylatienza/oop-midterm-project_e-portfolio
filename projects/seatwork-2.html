Seatwork COMP 009 · OOP · 2025

Seatwork 2 — Input/Output

This activity involves using predefined Java I/O classes to handle user interactions, specifically capturing student data and financial information through the console. It focuses on processing real-world scenarios, such as calculating tuition payments and determining scholarship eligibility based on input values.

Java Scanner Class User Input If-else Input Validation Memory Management

Overview

What this seatwork covers

The activity covers the various ways to capture user input in Java, primarily focusing on the Scanner class for parsing primitive types like integers and floats. It also touches upon the BufferedReader class for handled stream input and the Console class for secure data entry, while emphasizing the importance of closing these streams to avoid memory leaks.

Reflection

What I learned

Handwriting the code for this scholarship discount seatwork was a solid way to bridge the gap between user input and decision logic. Using the Scanner class to capture grades and tuition fees felt much more interactive than my previous projects, especially when implementing the if-else structures to determine if a student is eligible for a discount. Transitioning from C, I’ve noticed how Java’s nextLine() and nextDouble() methods make parsing different data types a lot more streamlined, though I have to stay disciplined about closing the scanner to prevent memory leaks. It was a lowkey challenge to ensure the logic flowed correctly from input to calculation, but it really reinforced how important input validation is for functional applications like a payment system. Overall, seeing how the JVM takes these inputs and processes them through the main method's parameters helped me appreciate the practical, "write once, run anywhere" efficiency of Java in a school setting.
